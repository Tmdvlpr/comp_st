# CURRENT_UI_SPEC — CS Monitor AI (фактический спек реализованного UI)

Снято ПО КОДУ из `frontend/src`. Все строки интерфейса цитируются дословно (как в коде).
Проект: React 19 + Vite + TS + Tailwind v4 + Plotly (`plotly.js-basic-dist-min`) + TanStack Query + anime.js.
Операторский дашборд мониторинга аномалий газокомпрессорной станции. Агрегаты ГПА-1/2/3.

Источники: `App.tsx`, `main.tsx`, `types/index.ts`, `components/**`, `styles/globals.css`.

## 1. Каркас приложения (App shell)

### 1.1 Уровень маршрутизации (`main.tsx` → `Root`)
- URL-хэш `#/s/<id>` = выбранная станция; пустой хэш (`#/`) = **экран-обзор станций** (`StationOverview`).
- `Root` рендерит либо `StationOverview` (station == null), либо `<App>` (детальный вид станции).
- Обёртка: `ErrorBoundary` → `QueryClientProvider`.
- Внутри `App` вид переключается между **monitor** и **schema** (state `activeView`), плюс наложенные полноэкранные режимы (Двигатель, Сравнение, Киоск, Отчёт).

### 1.2 Layout `App` (monitor)
CSS grid, `App.tsx` `styleRootMonitor` + `rootGridStyle`:
- `gridTemplateRows: '36px 1fr'` — строка 1: тикер (высота 36px, во всю ширину, все колонки); строка 2: контент.
- `gridTemplateColumns` (monitor): `${sidebarCollapsed ? '48px' : '272px'} 1fr 300px` — **[сайдбар | центр | правая панель]**.
- В режиме **schema**: `gridTemplateColumns: '1fr'` — сайдбар и правая панель скрыты, центр на всю ширину.

Регионы:
- **Строка 1, все колонки** — `Ticker` (верхняя панель).
- **Колонка 1, строка 2** — `Sidebar` (только monitor).
- **Колонка 2, строка 2** — центр: `<SchemaPanel active=…>` (display:none когда неактивна) + `<main class="js-view-monitor">` (только monitor).
- **Колонка 3, строка 2** — правая панель `DetailPanel` (только monitor).

Первый фокусируемый элемент: скрытая ссылка-пропуск `Перейти к основному содержимому` (href `#main-content`).
ARIA live-region (`role="status"`) для новых крит.аномалий: `Новая критическая аномалия. Всего непринятых критических: {n}`.

### 1.3 Верхняя панель `Ticker` (`Ticker.tsx`) — высота 36px
Слева направо:
1. Пульсирующая точка `--ok` + **часы** (`<Clock/>`, `ru-RU` HH:MM, тик 10с).
2. `left`-слот (из App): иконка «сеть» с числом датчиков (`title="Датчиков на станции"`, значение `stats.total_sensors ?? sensors.length`); иконка-треугольник с числом аномалий (`title="Аномалий за период"`, значение `stats.total_anomalies ?? 0`).
3. Распорка (flex-1).
4. Кнопка журнала: если есть непринятые — красный пилл `{unackedCount}` (`--crit`, pop-анимация при росте), затем текст **`Журнал (J)`** и `↗`.
5. `right`-слот (из App): `Freshness` + разделители `Sep` + группа иконок-кнопок `IconBtn`:
   - **Мониторинг** (`title="Мониторинг"`, active при monitor)
   - **Схема КС** (`title="Схема КС"`, active при schema)
   - **Двигатель · 3D-режим** (`title="Двигатель · 3D-режим"`)
   - **Отчёт смены** (`title="Отчёт смены"`, только в monitor)
   - **Дежурный режим (K)** (`title="Дежурный режим (K)"`)
   - **Тема: {theme} (T)** (`title=\`Тема: ${theme} (T)\``, sun↔moon crossfade)
   - `StationSwitcher` (если станций >1)
   - **К обзору станций** (`title="К обзору станций"`, стрелка влево; только если задан `onBackToOverview`)

Переключатель видов = 3 иконки: **Мониторинг / Схема КС / Двигатель · 3D-режим** (последний открывает полноэкранный режим, а не заменяет центр).

### 1.4 Клавиатурные шорткаты (`App.tsx`)
`J` — журнал (toggle), `K` — дежурный режим (toggle), `T` — тема, `Esc` — закрыть drawer/kiosk. Не срабатывают в полях ввода.

## 2. Мониторинг / Overview (центральный `main`)

Вертикальный стек (`styleMainPanel`, padding 12/20, gap 12):
1. `StatsGrid` (KPI-карточки)
2. `PriorityBanner` (топ-событие)
3. Баннер активного kind-фильтра (условно)
4. Нижняя сетка `styleBottomGrid`: `gridTemplateColumns: 'clamp(280px,28vw,352px) 1fr'` — **[Тепловая карта | График]**.

### 2.1 StatsGrid — KPI-карточки (`Stats/StatsGrid.tsx`)
Сетка **7 колонок** (`repeat(7,1fr)`), карточки в порядке:

| key | Точный label | value (поле) |
|---|---|---|
| ml | `Стат. выброс` | `stats.ml_count` |
| neg | `Физичность` | `stats.neg_count` |
| frozen | `Датчик завис` | `stats.frozen_count` |
| roc | `Скачок ΔV` | `stats.roc_count` |
| seasonal | `Сезонная` | `stats.seasonal_count` |
| regime | `Смена режима` | `stats.regime_count` |
| cross | `Кросс-ГПА` | `stats.cross_count` |

- Карточка кликабельна → фильтрует датчики/аномалии по типу (toggle). Активная — рамка/фон `--accent`.
- Von Restorff: value>0 — «тревога» (крупнее число `--fs-2xl`, левый бордер 3px по severity), value=0 — «тихая» (`--fs-xl`, приглушённая).
- Порог crit: `value >= 10` → цветной бейдж **`КРИТ`** в углу + красный левый бордер/подсветка.
- Число «добегает» (animejs) при изменении.

**Данных о значениях по остальным полям `StatsResponse` (crit_count/warn_count/info_count/ok_count/drift_count/regime_count/seasonal_count/roc_count) в самих карточках НЕТ** — карточки только 7 перечисленных.

### 2.2 PriorityBanner (`PriorityBanner/PriorityBanner.tsx`)
Показывается, только если есть непринятое событие severity `crit`/`warn` (crit приоритетнее, внутри — новее). Иначе — `null`.
Кликабельная строка (`title="Перейти к датчику"`), содержимое слева направо:
- точка severity (пульс у crit), бейдж `badge-sev` (`SEV_LABEL[severity]`), `top.gpa`, `ruSensor(sensor_name)`, `—`, `KIND_LABEL[kind]`, `description` (обрезается), время `ru-RU` (dd.mm HH:MM).
Рамка: красная только для crit; для warn — нейтральная `--line`.

### 2.3 Баннер kind-фильтра (`App.tsx`, `styleKindFilterBanner`)
Появляется при активном `kindFilter`. Текст: `Фильтр: {KIND_LABEL[...] ?? kind}` + кнопка **`✕ Сбросить`**.

## 3. Тепловая карта (`HeatMap/HeatMap.tsx`)

Заголовок секции (в App): `Тепловая карта` + бейдж с числом ячеек `heatmapCells.length`.

- **Легенда** (4 записи, `SEV_LABEL_LONG`): `Суперважно` (crit), `Внимание` (warn), `Норма` (ok), `Наблюдение` (info). Квадратные свотчи.
- **Таблица**: строки = имена датчиков (`ruSensor(name)`, отсортированы), колонки = ГПА (`ГПА-{n}`, sticky-заголовок). Ячейки 28×24px.
- Ячейка окрашена `color-mix` по severity. Отсутствующий на ГПА датчик — пунктирная прозрачная ячейка (`title=\`{name} · ГПА-{n} · датчик отсутствует на этом агрегате\``).
- Ячейка кликабельна (выбор датчика), draggable (drag-drop на график), keyboard (Enter/Space). Выбранная — рамка `--accent` + scale 1.3. Hover — scale + brightness.
- title/aria ячейки: `` `${ruSensor(name)} · ГПА-{n} · ${SEV_LABEL_LONG[sev]} · ${anomaly_count} аномалий` ``.
- Скрытый caption таблицы: `Тепловая карта критичности датчиков`.
- **Состояния**: пустой массив → таблица без строк (пустая). Скелетона нет (данные грузятся через query).

## 4. Карточка графика датчика (`Chart/SensorChart.tsx`) + шапка (`App.tsx`)

Область графика — зона приёма **drag-drop** датчиков (наложение доп. рядов). При drag: outline `2px dashed --accent` + оверлей **`Отпустите — наложить датчик на график`**.

### 4.1 Шапка карточки (App, `styleSensorHeaderRow`)
Если датчик НЕ выбран — плавающая кнопка **`⇄ Сравнить датчики`** (открывает ComparePanel), `title="Сравнить датчики (разные ГПА/типы) на одном канвасе"`.

Если выбран — слева блок идентификации:
- Имя `ruSensor(sensor.name)` (fs-md, 600)
- `sensor.tag` (monospace)
- `ГПА-{n} · {subsystem}` (monospace, приглушённый)

Справа тулбар (`styleSensorToolbar`):
- `от` `<DatePicker>` `до` `<DatePicker>` [`✕` сброс диапазона если задан]
- разделитель
- пресеты дней: **`1д 3д 7д 14д 30д`** (`[1,3,7,14,30]`); активный подсвечен `--accent`; недоступная глубина приглушена (opacity 0.45)
- разделитель
- **`★ Важные признаки`** (`title="Важные признаки аномалии: интерпретация (база кейсов) + графики параметров-вкладчиков. Совет: правой кнопкой выделите участок графика — откроются топ-5 вкладчиков по нему"`)
- **`⊕ Сравн. ГПА`** (toggle наложения того же датчика с других ГПА; `title="Наложить тот же датчик с других ГПА (сравнение, кросс-ГПА)"`)
- **`⇄ Сравнить`** (открывает ComparePanel)

### 4.2 Панель сфокусированного события (App, `js-focus-panel`)
Показывается при клике по событию этого датчика. Von Restorff: `--surface-3`, левый бордер 3px по severity.
Содержимое: бейдж `SEV_LABEL[severity]`, `KIND_LABEL[kind]`, время (`fmtStation`, dd.mm HH:MM), `Значение: {value.toFixed(4)}` [`· Отклонение: {deviation.toFixed(4)}`], справа: `✓ Принято` (если acked) или кнопка **`✓ Принять`**, кнопка `✕` (`aria-label="Закрыть панель события"`).

### 4.3 Чипы наложенных (dropped) датчиков (App, `styleDroppedChip`)
Ряд пунктирных чипов `{ГПА-n · имя}` с кнопкой `✕` (`title="Убрать с графика"`, hover → `--crit`).

### 4.4 Слои графика (Plotly, `SensorChart.tsx`)
Кривые/области (в порядке отрисовки):
1. **`Значение`** — факт, сплошная `#CDD9E5`, width 2.
2. **`Модель (MAE: {mae.toPrecision(3)})`** — прогноз, пунктир `#58A6FF`, width 1.5, `connectgaps:false`.
3. **`Дов. интервал`** — доверительный коридор: пунктирные границы `rgba(88,166,255,…)` + диагональная штриховка `fillpattern shape:'/'`; отдельными непрерывными сегментами (без «перепрыгивания» разрывов).
4. **Наложения** (overlaySeries): пунктир (`dash:'dot'`), палитра `OVERLAY_COLORS`; перетащенные (secondary) — на правой оси Y2; кросс-ГПА — на основной оси. Подпись: `ov.label` или `ГПА-{n}`.
5. **Маркеры аномалий** — по типам (`byKind`), цвет `KIND_COLOR`, форма `KIND_SYMBOL` (ml=diamond, frozen=circle, neg=star-diamond, roc/seasonal=triangle-up, regime=star, cross=triangle-down). Имя = `KIND_LABEL[kind]`. При активном фильтре подсвечен только один тип (крупнее, белая обводка), прочие opacity 0.15.
6. **Вертикаль «Начало мониторинга»** — линия на `train_ts` (`rgba(132,204,22,…)` dot) + аннотация текстом `Начало мониторинга`.
7. При активном kind-фильтре — мягкие прямоугольные подсветки кластеров этого типа.

Ось X формат `%d.%m %H:%M`, hover `%d.%m.%Y %H:%M`. Легенда — своя (Plotly-legend `orientation:'h'` сверху, но текст берётся из имён трейсов). hovermode `x unified`.

**Кастомный ховер (bklit-стиль)**: собственная вертикальная пунктирная линия-курсор, точки-маркеры на линиях, плавающий тултип (только «Значение»/«Модель», значения `.toFixed(4)`), пилюля с датой на оси X (`dd.mm.yyyy HH:MM`). Маркеры аномалий и коридор из тултипа исключены.

- разделитель
- `⌫` — **Очистить рисунок**

Фиолет `#C77DFF`, заливка + маркеры выше порога, пунктир-порог с аннотацией `порог новизны`. Высота ~116px.

aria-label области: `` `График датчика {name}: значение и прогноз модели, {n} точек` ``.

## 5. Правая аналитическая панель (`Detail/DetailPanel.tsx`)

Ширина колонки 300px. Заголовок: `Аналитика · {ruSensor(name)}` (или `Аналитика · —` без датчика).

**Вкладки** (`role="tablist"`, стрелочная навигация): `Здоровье`, `Качество модели`, `Дрейф`, `Калибровка`, `Доменные индексы`.

Пустое состояние (нет датчика): `Выберите датчик в сайдбаре слева или на тепловой карте`.

### 5.1 Вкладка «Здоровье»
- **Gauge**: крупная подпись `SEV_LABEL[severity]`, цвет по severity; sub: `{anomaly_count} аномалий за 24ч · {anomaly_count_30d ?? cnt30} за 30д`.
- Метрика `Текущее значение` = `sensor.cur`.
- Секция `Коды здоровья (по событиям)`: `ml (выброс)`, `roc (скачок)`, `drift (дрейф)` — счётчики по событиям.
- Секция `История severity · 30 дней` `(клик: событие дня / переход к дню)`: 30 клеток-дней (цвет по макс. severity дня; `tlColors` = ok/warn/teal/crit). Клик открывает событие дня или наводит график на день. Подписи под лентой: `−30д` … `сегодня`.
- Секция `Последние события датчика`: до 5 строк [время · точка · `KIND_LABEL[kind]` · `{value} (Δ{deviation})`]. Пусто: `Событий нет.`

### 5.2 Вкладка «Качество модели»
Метрики: `Модель (прод)` (`best_model` / `—`), `MAE (holdout)`, `nMAE` (+ бар, зелёный <0.05 иначе жёлтый), `R² (честный, holdout)`, `R² (in-sample)`.
Блок **`Как читать`** с пояснениями по каждому (Модель — «в проде один **CatBoost**…», MAE, nMAE, R² честный, R² in-sample).

### 5.3 Вкладка «Дрейф»
Если нет данных: `Нет данных дрейфа для этого датчика (недостаточно точек на рабочем режиме).`
Иначе Gauge: `ДРЕЙФ ↑` / `СТАБИЛЬНО`; метрики `Тренд остатка (7 дн)`, `Reversibility`, `CUSUM`, `Page-Hinkley` (значения `сработал`/`нет`/`—`). Блок `Как читать` (Вердикт / Метрика дрейфа / Тренд / Reversibility / CUSUM / Page-Hinkley).

### 5.4 Вкладка «Калибровка»
Если нет данных: `Нет данных калибровки для этого датчика.`
Метрики: `Коридор conformal (α=1%)`, `Порог POT-EVT`, `n_sigma` (`… (калибр.)`/`… (по умолч.)`), `Покрытие нормой` (%), `Доля алармов` (%). Блок `Как читать` + примечание `Пороги — в единицах остатка датчика. Калибровка на свежей норме holdout (robust σ через MAD), не на обучающих данных.`

### 5.5 Вкладка «Доменные индексы»
Заголовок `Наблюдаемые индексы — текущее значение и вариации`. 10 карточек `IdxCard` (порядок `INDEX_ORDER`): Политропный КПД η_p, Политропный напор H_p, Нагрев в нагнетателе ΔT, Перепад на АВО (range), Approach АВО, Shaft mismatch (ТНД=f(ТВД)), Остаток СТ (СТ=f(ТВД,ТНД)), Отношение оборотов ТНД/ТВД, Удельный расход топлива, ΔT подшипников.
Каждая карточка: имя, значение+единица (реальное из `sensor.domain` или дефолт), `норма: …`, интерпретация `now`, блок `ВАРИАЦИИ / ЧТО ЗНАЧИТ ОТКЛОНЕНИЕ:` (список условие→смысл).
Примечание внизу — зависит от наличия реальных данных (два варианта текста).
