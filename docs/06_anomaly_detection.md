# 06. Обучение и выявление аномалий

## 6.1 Идея

Для каждого датчика обучается модель «ожидаемого значения»: она предсказывает датчик по другим датчикам того же ГПА и по собственной короткой истории (лаги). Расхождение факта и прогноза + набор статистических правил дают аномалии. Это модель **физической согласованности**: датчик считается аномальным, когда ведёт себя не так, как «договорились» остальные сигналы и его недавнее прошлое.

## 6.2 Обучение (`train_and_save_models.py`, офлайн, однократно)

1. **Маппинг тегов** из БД (`build_tag_mapping`): `GPA-1.GPA-1.PD.PV → gas_pressure_out_gpa__GPA1`.
2. **Загрузка** всей истории (или до `--cutoff-date`) в wide-формат; исключаются boolean-датчики (`is_*`).
3. **Фильтр простоев:** по оборотам (rpm) стоп-строки → NaN, +30 мин прогрева после пуска отбрасываются. Так `sensor_range` и пороги считаются по «рабочему» режиму.
4. **Признаки** (`_make_features`): другие датчики ГПА (предикторы) + `lag1/2/3` целевого + `roll6 mean/std` (скользящее за 30 мин, только прошлое).
5. **Сплит и обучение:** последние 20% — валидация; CatBoost `RMSEWithUncertainty` с early stopping; затем финальный `fit` на 100% данных с `best_iter`.
6. **Метрики:** `r2` (на 100% данных — in-sample), `residual_std_val` и `residual_mean_val` (std и среднее |остатков| на валидации = фактически валидационная MAE), `sensor_range` (перцентиль 99–1).
7. **Сохранение:** `models/<station>/<sensor>.cbm` + единый `metadata.json`.

**Гиперпараметры:** `iterations=800, depth=6, lr=0.05, RMSEWithUncertainty, subsample=0.8, l2_leaf_reg=20, seed=42`. Глобальные: `ANOMALY_N_SIGMA=5.0`, `MIN_BUFFER_PCT=0.15`, `VAR_SMOOTHING=24`, `VAL_FRACTION=0.2`.

### Состав `metadata.json`
`last_train_timestamp`, `anomaly_n_sigma`, `min_buffer_pct`, `var_smoothing`, `gpa_ids`, `tag_to_name`, `name_to_tag`, и по каждой модели: `predictors`, `feat_cols`, `tag`, `gpa_id`, `r2_train`, `residual_std_val`, `residual_mean_val`, `best_iterations`, `sensor_range`, `model_file`.

### ⚠️ Аудит метрик (важно)
- `mae_train` читается API и дашбордом, но **обучением не пишется** → MAE везде `0.00` (мёртвая метрика). Валидационная MAE по сути есть — это `residual_mean_val`.
- `r2_train` — **in-sample** (на 100% данных после финального refit), оптимистичен.
- Этот же R² управляет шириной коридора аномалий (`min_abs_pct`).
- **План (фаза 5):** считать `mae_val/rmse_val/nmae_val`, заполнить `mae`, перевести пороги и UI с R² на (нормированную) MAE. Подробности и фазы 6/7 (честная валидация, робастность, калибровка) — в `PROMPT_PLAN_cs4_anomaly_health_journal.md`.

## 6.3 Детекция (`live_predict.py`, онлайн, непрерывно)

**Онлайн-цикл (`run_continuous`, каждые 5 мин):**
1. первый проход — история до `MAX_HISTORY_DAYS=30`; далее — только свежий срез (`fetch_latest_slice_from_db`), добавляется в память, старое подрезается;
2. wide-преобразование + флаги работы ГПА (`__running_GPAn`);
3. прогноз и детекторы по каждому датчику (`predict_sensor`);
4. кросс-ГПА и смена режима (`run_once`);
5. запись `anomalies` (БД) + `state.json` + HTML-дашборд (`_write_live_state`, `build_dashboard`).

**Инвариант онлайн:** аномалии фиксируются только при `is_live` — время точки **позже** `last_train_timestamp`. То есть детекция работает на **текущих** данных в реальном времени, а не только на истории. Бэкафилл истории — отдельная разовая операция (план, фаза 4), он не подменяет онлайн-цикл; единый запуск (фаза 12) поднимает именно `run_continuous`.

### Коридор нормы (для ML-детектора)
`y_std` — из неопределённости CatBoost (сглажена, `VAR_SMOOTHING`), не ниже `residual_std_val`. Полуширина:
`hw = max(n_sigma·y_std, sensor_range·min_buffer)·transient_boost`, не ниже `min_abs_error` (зависит от R² — см. план фазы 5) и небольшого относительного пола. `transient_boost` расширяет коридор на быстрых переходных процессах.

## 6.4 Словарь и 7 детекторов

| Код | kind | Детектор | Severity | Логика (кратко) |
|----|------|----------|----------|-----------------|
| 1 | `ml` | Стат./ML-выброс | crit | `|факт−прогноз| > hw`, при `is_live`, отклонение значимо; подавлен на стоянке/в прогреве |
| 2 | `neg` | Сбой физичности | crit | отрицательное значение у неотрицательных датчиков (давление/поток/вибро/газ), 2 точки подряд |
| 3 | `frozen` | Залипание | warn | ≥12 одинаковых значений подряд (60 мин); на стоянке не считается |
| 4 | `roc` | Скачок скорости | warn | `|Δ| > порог` по типу датчика (rpm 10%…газ 40%), с анти-спайком; не на переходах |
| 5 | `seasonal` | Сезонная | info | отклонение от почасового профиля (медиана/σ по часам, blend train/all) > 3σ_h |
| 6 | `regime` | Смена режима | info | синхронный дрейф группы сигналов ГПА (по rpm и доле «движущихся» датчиков) |
| 7 | `cross` | Кросс-ГПА | info | z-score датчика относительно тех же датчиков на других ГПА > 2.5, подтверждение 15 мин |

Источник правды словаря — `anomaly_types.py` (`CODE_TO_KIND`/`KIND_TO_CODE`/`KIND_SEVERITY`). Соглашение `0=OK` используется для `health`.

## 6.5 Подавление ложных срабатываний
- **Стоянка ГПА** (`__running_GPAn < 0.5`): ml/frozen/roc/seasonal подавлены (датчики константны, модель обучена на работе). Загазованность (`gas_leak`) мониторится всегда.
- **Прогрев** (+30 мин после пуска): пропускаются только грубые отклонения.
- **Переходы пуск/останов** (±2 точки): roc/cross подавлены как легальные скачки.
- **Смена режима:** в окне ±15 мин ml/roc/seasonal гасятся (кроме очень крупных остатков).
- **fail-open:** нет статус-тега → считаем «работает» (риск ложных на стоянке, см. аудит D3).

## 6.6 Эпизодизация и события
В `_write_live_state` true-тики с разрывом ≤2 тиков (10 мин) объединяются в один эпизод-событие (`timestamp..ts_end`, `points`, пиковое значение/отклонение). Severity карточки датчика — по последним 24 ч; полный счётчик за окно — `anomaly_count_30d`. Эти же эпизоды → строки `anomalies` (а планом — и `health`, и журнал уведомлений).

## 6.7 Дрейф моделей
`model_drift`: если медиана |остатков| за 7 рабочих дней > 2× `residual_mean_val` — модель помечается как «дрейфующая». Сейчас может рекомендовать переобучение. **Решение проекта:** переобучение не предусмотрено — план делает этот сигнал чисто информационным (фаза 5).
