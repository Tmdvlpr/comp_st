### A1. Гейт `running` внутри `predict_sensor` (~25 строк, сигнатуру не менять)

**Файл/место:** `live_predict.py`, `predict_sensor`, вставка после `df_t = df_t.dropna()` (строка 240) и применение к маскам (строки 289, 302, 315, 321, 327).

**Суть.** `df_wide` уже передаётся в функцию — достать флаг прямо из него:

```python
gpa_num = target.rsplit('__GPA', 1)[-1]            # 'rpm_st__GPA1' -> '1'
run_col = f'__running_GPA{gpa_num}'
if run_col in df_wide.columns:
    running = df_wide[run_col].reindex(df_t.index).ffill().fillna(1.0)  # fail-open
else:
    running = pd.Series(1.0, index=df_t.index)
is_stopped = np.asarray(running < 0.5)

# grace 30 мин (6 точек) после пуска 0->1
restart = (running.diff() > 0.5)
warm = np.asarray(restart.rolling(6, min_periods=1).max().fillna(0) > 0)

is_gas_leak = 'gas_leak' in target.lower()   # загазованность мониторим всегда
suppress = is_stopped & ~is_gas_leak
```

Применение (точечно, по одной строке на маску):
- `ml_mask = ml_mask & ~suppress`; в прогрев — ослабленный порог: `ml_mask &= ~(warm & (resid <= hw * 2.0))`. Прогрев важен ещё и потому, что lag1-3/roll6-фичи первые 30 мин после пуска загрязнены стоповыми значениями.
- `frozen_mask &= ~np.asarray(is_stopped)` — глушим и для gas_leak тоже? Нет: для gas_leak оставить как есть (газоанализатор работает и при стопе, его залипание — реальная неисправность). Т.е. `frozen_mask &= ~suppress`.
- `roc_mask &= ~np.asarray(suppress | transition)`, где `transition` = ±2 точки вокруг любого фронта `running.diff() != 0` — пуск/останов сами дают легальный скачок.
- `seasonal_mask &= ~suppress`.
- `is_negative` — см. Б2 (гейт + deadband).

**Риск:** статусный тег `STATES_GTD.5` пропал/запаздывает → `fillna(1.0)` (fail-open: лучше лишнее срабатывание, чем слепота). Обратный риск — статус 0 при реально работающем ГПА: пропустим аномалии; митигировать (фаза 2) эвристикой `running = max(status, rpm_tvd > 0.05*sr_rpm)`.

**Проверка:** реплей `run_once(existing_df=None)` на текущей 30-дневной истории (там есть останов ГПА-1), diff счётчиков `live_state.json -> events` по `(gpa, kind)` до/после. Ожидание: GPA1 frozen 299→~0, neg 102→~0-5, seasonal 97→~0; счётчики GPA2/GPA3 не изменились (это регресс-критерий).

Сам остановленный датчик после `.mask()` даёт NaN → `fillna(False)` на строке 2269 уже отсекает его.
