# CS Monitor AI — передача задачи (2026-07-03)

Онбординг для новой сессии, продолжающей работу над системой мониторинга ГПА (48 датчиков × 3 ГПА,
5-мин телеметрия, станция ohangaron, прод v23, cutoff 2026-05-25).

## Контекст: в чём была проблема
Система выдавала россыпь ложных «аномалий» на вибрации. Диагноз (доказан на данных):
вибро-модели предсказывались по ГАЗОВОМУ ТРАКТУ → R² от −0.98 до −17.6 (хуже константы) →
смещённый центр коридора → ложные пробои вверх. Плюс система не отличала **смену режима** от
**дефекта**. Полный разбор: `backend/AUDIT_2026-07-03.md`.

## Что сделано этой сессией

1. **Аудит всего бэкенда** (4 агента) → `backend/AUDIT_2026-07-03.md`. Ключевое: `conservative_q=0.5`
   обнулял block-bootstrap (покрытие ниже 98%); baseline-рассинхрон пускал R²<0 модели в ml_corridor;
   pre_fault не гейтил калибровку; self-band строился выборочно; сбой чтения БД маскируется как «нет
   данных»; shaft-baseline ре-фитится в live; флот-контраст вырождается на неполном флоте;
   aging_watchdog не подключён.

2. **Тир-3: единый 5-классовый детектор** — `backend/detection_v2.py` (НОВЫЙ, самодостаточный).
   Лестница: не-физ | залипание | выброс | смена режима | дефект. Доказан на реальном кэше по всем
   классам, включая разрешение случая С4 (согласованный сдвиг) через флот-контраст. НЕ ВРЕЗАН в live.

3. **Правки `backend/train.py` (метрика MAE, R² убран из решений):**
   - `conservative_q` 0.5→0.9 (константа `DEFAULT_CONSERVATIVE_Q`) — автокорреляция расширяет q̂;
   - baseline = healthy-медиана (не train-среднее); `beats`/качество по MAE;
   - self-band строится ВСЕМ (`build_self_candidate(force_build=True)`); модель служится только если
     бьёт self по MAE (`_self_band_mae`), иначе self-band основной — даже под FORCE;
   - `pre_fault` ГЕЙТИТ калибровку (suspect-режим → univariate_only);
   - признаки вибро = соседние вибро-каналы + рабочая точка (не газовый тракт).

4. **Независимый ревью правок нашёл 5 проблем — ВСЕ ИСПРАВЛЕНЫ и проверены (compile+smoke):**
   - #2 mae_self-leakage: центр для роутинга теперь строго ≤cutoff, тот же eval-support, min≥30;
   - #1 pre_fault: гейт по ВЕЛИЧИНЕ тренда (`mag_frac`), не только z (иначе массовая демотация при n=6000);
   - #4 conservative_q: FORCE ×1.5 базируется на q_naive (не двойная консервативность);
   - #3a вибро: при <2 кросс-фич — self-band-only, датчик НЕ выпадает молча;
   - #5 detection_v2.episodes: плотность-в-окне (перемежающийся дефект больше не подавляется).

5. **Фронт: единая корпоративная дизайн-система** (артефакты claude.ai). Токены: титан (нейтраль) +
   оранжевый #FF8C1A (бренд/действие) + 5 классов (состояние) + медиа (трубы); Oswald+Inter+mono.
   Навигация: флот → агрегат → подсистема → датчик + рабочий список эпизодов.

6. **Чистка репо:** −1.3 ГБ (19 лишних model-каталогов, `_archive`), удалён мёртвый код.
